Section 2 — Dead Ends

States where the user is stranded with no forward navigation.

  • Dead EndBonsai ceremony "Later" button The planting ceremony overlay (_bonsaiPlantingCeremony) is shown after seeds are awarded in non-onboarding mode. The "Later" button calls this.closest('.bonsai-ceremony-screen').remove(); transitionTo(() => navigate('profile')). This correctly routes to Profile, so this is not a pure dead end — but it dismisses the ceremony permanently and does not open the Rewards page or Bonsai overlay, so the user may not know where to find their garden. Minor UX gap but not a blocker.
  • Dead EndAuth reset view (#auth-reset-view) The password reset form is only shown when ?token= is present in the URL. Users who arrive via a direct link land correctly; but if they navigate away or close the modal without completing the reset, there is no way to re-trigger the view without the original email link. After a successful reset the view auto-hides to the login form after 2.5 s.
  • Dead EndLevel unlock screen (after level 2 or 3 unlock) showLevelUnlockScreen() renders a full-screen particle overlay. The sole dismiss button calls screen.style.opacity = '0'; setTimeout(() => { screen.style.display = 'none'; navigate('techniques'); }, 600) — navigates to Techniques. This is now fully wired. No dead end.

Previously listed dead ends now resolved: Bonsai plant view (no pot) — has "Back to Profile" button. Bonsai plant view (no seeds) — has "Back to Profile" button. Level unlock screen — navigates to Techniques on dismiss.

Section 3 — Inaccessible Screens

Elements defined in code or HTML that cannot be reached through normal app navigation.

  • OrphanAuth reset view (#auth-reset-view) The password reset form inside the auth modal is only shown when the server detects ?token=… in the URL on page load. It is never reachable via any in-app button. Users must arrive via the emailed reset link.
  • OrphanPro paywall modal (#paywallModal) Shown programmatically by requirePro() when a Free-tier user attempts a Pro action (My Plan, Ask). Not directly accessible from any visible nav element.
  • OrphanCost confirmation modal (#costModal) Shown programmatically by confirmApiCall() before each AI API call. Shown automatically; not reachable via any button in normal browsing.
  • OrphanDiagnostic overlay (#diagnosticOverlay) Opened by openDiagnostic() from within the Missions block inside the Profile accordion, which itself requires auth. Missions are only available to logged-in users, and the accordion is collapsed by default.
  • OrphanQuit session modal (#quitSessionModal) Only shown by openQuitModal() when a user attempts to close an active session (guest pacer, pro pacer, or MoB overlay). Not reachable without an active session.
  • OrphanAsk modal (#askModal) Opened by openAskModal() from a technique or meditation detail view "Ask" button. Requires Pro tier. The button is only visible when a detail view is open.
  • OrphanSession edit sheet (#sessSheet) Bottom sheet for editing cycle count, opened by sessionEditCycles() from the live session overlay. Only accessible during an active session.

Previously listed inaccessible items now resolved: #achievements-accordion-wrap — removed from HTML. #profile-pro-banner — removed from HTML. Session intro duration tiles — now wired to pro pacer via _closeSessionIntroWithDuration().

Section 4 — Unwired / Inert Click Events

Buttons that exist in the UI but do nothing useful or fire silently.

  • UnwiredBonsai dev water button (bonsaiDevWater) bonsaiDevWater() exists in bonsai.js and bypasses the one-water-per-day guard to advance cycles instantly. The function is present in production code with no guard (no if (DEV) check). If a button calling it exists in the HTML bonsai overlay, it is unguarded in production.
  • UnwiredWaitlist email — not actually sent to a server submitWaitlistEmail() saves the email to localStorage only. It never makes a network call to a mailing list service or the Flask backend. The user sees a success state but the email is not stored anywhere persistent or sent anywhere.
  • UnwiredTriad Lab feature cards (Missions, My Plan, AI Coach) The three cards inside the Triad Lab section display status labels ("In Beta", "Coming Soon") but are not interactive. They have no onclick handlers and cannot be tapped to navigate anywhere.
  • UnwiredReading list "Mark read" toggle (toggleRead) toggleRead() exists and saves to localStorage + queues a sync. However the reading list is rendered inside a display:none wrapper in the accordion — it is invisible to the user. The function is live code but the UI element that triggers it is hidden.

Previously listed unwired items now resolved: Session intro duration tiles — fully wired to _closeSessionIntroWithDuration() which calls proSelectTile() and _proStartCountdown().
