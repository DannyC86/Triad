Section 2 — Dead Ends

Screens or states the user can reach but cannot leave through normal UI — they must close the overlay manually or use the OS back gesture.

  • Dead endBonsai Plant view — no pot yet Shows "earn a pot from your first breath session" but has no button and no back/close button visible. The overlay header has a close button but it is not immediately obvious. User must tap the close X at top right.
  • Dead endBonsai Plant view — pot earned, no seeds yet Shows "earn seeds from your first meditation" but has no action button. Same issue as above — user must find the close X.
  • Dead endLevel unlock screen "Start Practising" only fades and hides the overlay — no navigation follows. User stays on whichever section triggered it with no contextual next step.
  • Dead endBonsai ceremony "Later" button Removes the ceremony popup but navigates nowhere. User is left on Profile with no indication of what to do next.
  • Dead endOnboarding meditation results — "Explore the App" button Calls closeMobSession() + navigate('home'). Lands on Home but the home screen at this point has no strong prompt for what to do next — no missions card, no "continue" call to action.

Section 3 — Inaccessible Screens

Screens defined in code that a user cannot reach through any normal navigation path.

  • InaccessibleRewards page — empty state (for new users) The Rewards nav button always shows on Profile regardless of bonsai unlock state. A brand-new user can tap it and see the empty state, but the empty state has no link to explain how to unlock rewards or to start a breath session.
  • InaccessiblePlan section for non-Pro users The Plan nav tab is visible and tappable, but every action inside it (Generate plan, send message) is gated behind Pro. A non-Pro logged-in user tapping Plan reaches a screen where nothing works — they see the form but hit the paywall immediately on any attempt.
  • InaccessibleAsk modal for non-Pro users The "Ask a question" button on technique/meditation detail views only appears after the modal is opened and requires Pro. Non-Pro users cannot use it.
  • Inaccessible#achievements-accordion-wrap This element is permanently hidden (display:none) — it is a legacy DOM stub kept for JS compatibility. It cannot be reached or seen by users.
  • Inaccessible#profile-pro-banner Permanently display:none — element kept for legacy JS calls. Never shown to users.
  • InaccessibleSession intro popup duration selector (phase 2 tiles) The duration tiles (1/2/5/10/20 min) in the pre-session intro popup set _siSelectedDuration but this value is never passed to the session. The pro pacer has its own length selector. These tiles are visible but functionally inert.
  • InaccessibleAuth reset-password view Only shown when the page loads with a ?token= URL parameter from a password reset email. Cannot be reached through in-app navigation.

Section 4 — Click Events Not Yet Wired

Buttons that exist in the HTML but call a function that doesn't do anything meaningful, or where the wiring is missing.

  • InertSession intro popup — duration tiles siSelectDuration(this) is called on each tile click and sets _siSelectedDuration, but _closeSessionIntroWithDuration() ignores this value. The selected duration has no effect on session length.
  • InertBonsai dev water button (#bonsaiDevBtn) Calls bonsaiDevWater() which bypasses the one-water-per-day limit. This is a development helper that is permanently visible in the UI (no dev-mode guard). Should be hidden from production users.
  • InertPro pacer "Infinite" toggle in edit-cycles sheet (#sessSheetInfinite) Calls sessToggleInfinite() which exists in session.js, but the infinite mode only applies to the generic session overlay, not the pro pacer. The button is visible in the sheet but has no effect during a pro pacer session.
  • PartialWaitlist submit button submitWaitlistEmail() saves to localStorage only — no API call. Confirmation is shown in-modal but the email is never actually sent anywhere.
  • PartialAuth modal — "Continue browsing free →" link Calls closeAuthModal() only. Expected to also navigate somewhere contextual but currently just closes the modal with no further action.
