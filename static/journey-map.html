Dead Ends, Orphans & Conflicts

Dead Ends — screens with no exit route

  • Dead End#bonsaiOverlay PlantView (no pot) — Shows "earn a pot" message, no action button, no back nav. User must use OS back or tap outside. No closeBonsaiScreen() call is wired.
  • Dead End#bonsaiOverlay PlantView (pot, no seeds) — Same issue; shows "earn seeds" message but no action and no exit button.
  • Dead End#levelUnlockScreen — #lusBtn only closes the overlay; no navigation to anywhere. User stays on whichever section triggered it with no contextual prompt.
  • Dead EndBonsai ceremony "Later" btn — Removes the ceremony DOM node but provides no forward navigation. User is left on whichever section they were in.

Orphaned Functions — defined but never called from UI

  • OrphanupdateFab() (navigation.js:624) — Declared as no-op: "action bar removed". Called in 4 places (showTechniqueDetail, hideTechniqueDetail, showMeditationDetail, hideMeditationDetail). Safe to remove all 5.
  • OrphanhighlightLibraryEntry(slugId) (session.js:557) — Marked "Backwards-compatible alias for legacy callers." Not called from any UI element or other function in these files. Dead alias.
  • Orphan_mobGoToPage2() (navigation.js:1855) — Marked "Legacy alias" for _mobGoToPage2Direct(). Not called from anywhere.
  • OrphancloseStartSession() (navigation.js:601) — Calls sessionCancel(). Not wired to any button or caller in these files.
  • OrphancollapseP(text, uid) (navigation.js:198) — Ignores uid param; just wraps in <p>. Called nowhere in read files.
  • Orphan_siSelectedDuration (navigation.js:543) — Set by siSelectDuration(tile) but never consumed by _launchSession(). The SessionIntroPopup duration selector has no effect on session length in the Pro Pacer (it has its own length picker).

Multiple / Conflicting Routes to the Same Screen

  • Multi-Route#sessionOverlay completion screen — Reached by 3 different paths: _pacerSessionEnd() (guest pacer), _proSessionEnd() (pro pacer), _sessComplete() (generic session). All three manually set #sessionView.classList.remove('active') and #sessionComplete.classList.add('active') inline rather than calling a shared function. The completion button labels (scDuration, scCycles, scBreaths) are set differently by each path.
  • Multi-RouteopenMobSession() — onboarding mode — Reachable with _mobIsOnboarding=true from two paths: (a) post-breath completion via _sessionCompleteGo('meditate'), and (b) _sessionCompleteGo('meditate') called again from any future breath completion. This means the 30s fixed session and hidden cancel buttons persist for every "Meditate Now" clicked from any session completion, not just first-time onboarding.
  • Multi-RouteTechnique detail — Reachable via: nav bar → #techniques card; profile completion tile; profile session history item; session picker; AI chat practice link; related card on another technique; library breathwork tab; profile favourite practice strip. All route correctly but some call navigate('techniques') first (resets list view) while openLibraryEntry uses keepDetail:true.

No-ops / Unimplemented Referenced Functions
